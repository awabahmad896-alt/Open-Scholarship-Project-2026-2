# Frontiers of Memory: A Data Geography Analysis of Roman Imperial Heritage

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.20836645.svg)](https://doi.org/10.5281/zenodo.20836645)

**Open Scholarship Project 2026**

Welcome to my project! This repository explores how the Roman Empire is digitally remembered and preserved today. By looking at metadata from Europeana (an open cultural heritage platform), this project asks a simple but important question: 

**Which modern countries and institutions are the primary digital custodians of Roman Imperial heritage, and how does the quality of their data compare?**

## The Dataset

The data for this project comes from the [Europeana API](https://api.europeana.eu). I searched for cultural heritage objects tagged with "Roman Empire" and requested the "rich" metadata profile. The resulting dataset gives us a great look at what European institutions have digitized, with no strict time limits on the age of the objects themselves. 

If you just want the final, clean data without running the code, it is permanently archived and free to download via the Zenodo DOI badge at the top of this page!

## The Workflow

Here is a quick summary of what the code actually does:

1. **Data Collection:** It loops through the Europeana API grabbing thousands of records and saves them as a raw CSV. 
2. **Cleaning:** The raw data originally had 631 columns (many of them empty or messy). The code filters this down to just the 47,157 most useful records and 9 key columns. 
3. **Enrichment:** I created two new variables. First, a `quality_category` that groups Europeana's 0–10 "completeness score" into Simple Low/Medium/High tiers. Second, a `title_short` variable that trims down massive text strings so the data uses less computer memory.
4. **Analysis & Visualisation:** The code groups the data by country and institution, calculates who has the most records, and checks their average metadata quality. It then builds bar charts to show these comparisons clearly.

## Data Dictionary

If you are exploring the `processed` dataset, here is what the 9 columns mean:

| Variable | Description | Example / Values |
|---|---|---|
| `country` | The modern country where the museum/institution is located | `Germany`, `Italy` |
| `dataProvider` | The specific museum or library holding the object | `Rijksmuseum` |
| `language` | The language of the object's metadata | `en`, `de`, `fr` |
| `type` | What kind of object it is | `TEXT`, `IMAGE`, `3D` |
| `completeness` | Europeana's score for how detailed the metadata is | `0` to `10` |
| `quality_category` | My custom grouping of the completeness score | `low`, `medium`, `high` |
| `year` | The historical era or year from the metadata | `100 AD` |
| `title_short` | A shortened version of the object's title | String text |
| `score` | How relevant Europeana thought the object was to my search | `Float number` |

## Ethics & Limitations

Working with historical metadata is tricky, so it's important to be transparent about what this data *can't* do:

* **Coverage Bias:** This only includes institutions that actively share data with Europeana. Museums outside of Western Europe, or those with less funding for digital projects, are underrepresented.
* **Keyword Limitations:** The results depend entirely on someone typing the exact English phrase "Roman Empire" into the metadata. Objects described in other languages or under different historical terms might be missing.
* **What "Completeness" Means:** The completeness score just measures how many boxes the museum filled out on the digital form. It doesn't mean the historical information is perfectly accurate.
* **Privacy:** This dataset is just about old historical objects and the museums that hold them. There is zero personal or sensitive human data included.
